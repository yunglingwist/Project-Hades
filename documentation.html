Project Layout

Atakan: Home Page & Exhibition Page

Built the homepage as the thematic entry point to the exhibiton. The layout prioritized atmosphere: time-fitting palette, groovy music, our antagonist Zagreus introducing you into the narrative. In case a user might not the familiar to the concept of the game, brief info and embedded links to it is framed right below the narration, centered to pose as a warning. Right below is side scrolling text blocks to invite user to start exploring. Margins were widened to create breathing space and keep the content from touching the viewport edges. Responsive scaling was added so the title remains dominant regardless of device size. The homepage needed to instantly communicate theme and simplicity made that possible.

The Exhibition page was structured as the core navigation interface of the digital museum. Instead of relying on a standard menu, created a spatial layout inspired by the vertical map of the game. We chose to include both in-game map and museum exhibition rooms to reinforce the connection between the two worlds wtih only a swipe motion. Each exhibition room is represented as a node on the grid, positioned to mimic the in-game geography. Markers respond on hover, revealing the item associated with each exhibition area.

Also for smoother function, turned a theme-switcher into javascript. Later added to all pages located on right corner of the header for accesibility. And the page titles in the header turns into a hamburger menu for optimization through devices of all sizes.

Ceyda: Documentation Page, moodboard & stylization

Handled the documentation page, had to present text-heavy content while keeping it readable and non-static. A single-column layout was chosen for desktop screens: text on one side, visual development materials on the other (sketches, notes, iterations). For mobile, the layout collapses into a stacked sequence to maintain legibility. The goal was transparency — to show how the project was built, not only the end result. Placed on top of the page are three boxes containing each teammates picture and name, then three collapsible containers below containing main aspects of project development process. Collapsible text is for the users that might not be intrerested in such techincal details. Also to further ease heavy content, explanation and image matching theme is embedded in the css file of each theme. Simple animation is added for the viewer to notice the changes. Audio matching the theme of each era is implemented with the exception of Documentation page to avoid distraction while reading.

Also built the moodboards and set global style direction. We used Figma to visualize our first prototype. Altough the palette was inspired by ages, we agreed to focus mostly on dark themes as the game takes place in the underworld. CSS variables were later introduced to unify these style decisions across contributors. This prevented color drift and ensured that page-level visuals felt cohesive even though multiple people worked separately.

Yigit: Map Page & Disclaimer Page

The Map page required more technical layout solution. It needed to support many objects presented in repeatable format. Designed style-popup system using css: real object on the left, game item on the right, metadata inside boxes, expandable sections beneath. Flexbox was used instead of strict grid since it handles variable content better. Each box was assigned padding and maximum width constraints so images didn't stretch or overflow at large resolutions. A challenge was balancing text volume especially in historical & narrative descriptions so buttons — back/next, show more, +/- for language and content— were implemented and keep the layout visually level between entries. CSS grid was used to define a multi-cell area where each exhibition room could be positioned visually rather than linearly. Also to establish chronological timeline objects are manually places according to their historical metadata. This approach eased spacing and scale, allowing room names to occupy varying grid spans depending on their importance or volume of displayed material.

The map of the museum gives access to each item. Pop-up system is used with the intention of simulating walking through a real museum, where to see everything you have to get closer. The hover interaction helps guide users visually.

The disclaimer page required a informational tone specifying our purpose and copyrights. A minimal, single-column layout was chosen, with consistent margins and controlled line length to keep the legal text readable without distraction. Avoided heavy styling here on purpose reinforcing that the project interprets history through digital media rather than replacing museum authority.

Themes

Each member was assigned two chronological aesthetic eras to observe how classical mythology has been interpreted through time. The goal was to map historical continuity: how myth objects change in meaning across centuries, eventually resurfacing in modern media like Hades. For each theme we picked a font family, 3-5 key and accent colors, created a logo, a map and the antagonist to complete the aesthetic. As priorly mentioned, only light themes are available to the match game locations.

Metadata

Each team member wrote metadata entries on JSON files for 5–6 artifacts taking RSTU properties into consideration, linking museum objects to in-game equivalents. The structure for every entry included:

Historical description –material, origin, symbolism–; Game narrative –how the item transforms inside Hades–; Four writing levels –language simple/complex, content simple/complex, each divided into two for "show more" property–; Bibliographic web references for both museum and game sources.

The format allowed the exhibition to communicate on multiple cognitive layers — children, casual readers, and scholars can extract meaning from the same artifact.
